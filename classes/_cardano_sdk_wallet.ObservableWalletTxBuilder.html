Class ObservableWalletTxBuilder

TxBuilder concrete implementation, based on ObservableWalletTxBuilderDependencies. It relies on ObservableWalletTxBuilderDependencies for building, signing, sending, etc. Usage examples are in the unit/integration tests from builtTx.test.ts.

build

  • build(): Promise<MaybeValidTx>

  • Implementation of TxBuilder.build

    • Defined in packages/wallet/src/TxBuilder/buildTx.ts:138

    Builds a ValidTxBody based on partialTxBody. Performs multiple validations to make sure the transaction body is correct. In case validations fail, it creates a TxBodyValidationError instead of ValidTxBody.

    Returns Promise<MaybeValidTx>

    • In case it is a ValidTx, it can be used to sign and submit the transaction. This is a snapshot of transaction. Further changes done via TxBuilder, will not update this snapshot.
    • In case of it is an InvalidTx, it embeds a TxBodyValidationError with more details

delegate

  • delegate(poolId?: PoolId): TxBuilder

  • Implementation of TxBuilder.delegate

    • Defined in packages/wallet/src/TxBuilder/buildTx.ts:118

    Configure transaction to include delegation.

    • On build(), StakeKeyDeregistration or StakeDelegation and (if needed) StakeKeyRegistration certificates are added in the transaction body.
    • Stake key deregister is done by not providing the poolId parameter: delegate().
    • If wallet contains multiple reward accounts, it will create certificates for all of them.

    Parameters

    • Optional poolId: PoolId

    Returns TxBuilder
